NEW APPLICATION
===============

Create a new Django application called:

statement_engine


Purpose
-------

Convert Portuguese bank statement PDFs into the TOCOnline accounting format.

The system must:

1) Receive bank statement PDFs
2) Extract transaction data
3) Insert the data into a TOCOnline Excel template
4) Provide the processed file for download
5) Maintain a full processing history


IMPORTANT

Extraction logic already exists and will be injected later.

You only need to create the **architecture and processing pipeline**.

DATABASE MODELS
===============

Model 1: StatementTemplate

Purpose:
Store TOCOnline Excel templates uploaded by STAFF or ADMIN.

Fields:

id (UUID)
name
version
template_file (FileField)
active (Boolean)
uploaded_by (User FK)
created_at


Model 2: StatementJob

Purpose:
Represents one processing job.

Fields:

id (UUID)
owner (User FK)
template (FK StatementTemplate)
bank
original_pdf (FileField)
result_file (FileField)
status
created_at
processed_at
error


Model 3: StatementProcessingLog

Purpose:
Track pipeline steps for debugging and auditing.

Fields:

job (FK StatementJob)
step
message
created_at


PROCESSING PIPELINE
===================

The processing workflow must be:

Upload PDF
    ↓
Create StatementJob
    ↓
Send task to Celery
    ↓
Run processing pipeline

Pipeline steps:

1 Validate PDF
2 Extract transactions
3 Get active template
4 Clone template
5 Insert extracted transactions
6 Save result file
7 Update job status
